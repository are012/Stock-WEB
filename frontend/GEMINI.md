# 프론트엔드 개발 규약

## Next.js 특정 규약
- App Router를 기본으로 사용(Pages Router 아님)
- 서버 컴포넌트를 우선적으로 사용하여 초기 로딩 속도와 SEO를 최적화하며, 상태 관리가 필요한 곳에만 클라이언트 컴포넌트를 사용
- 이미지에는 next/image 컴포넌트를 사용하고, 폰트는 next/font로 최적화.

## 상태 관리 및 데이터 패칭
- 서버 상태: TanStack Query를 사용하여 백엔드(사용자 데이터) 및 AI 백엔드(실시간 매매 시그널, 투자 리포트)의 데이터를 효율적으로 캐싱하고 동기화
- 클라이언트 상태: Zustand를 사용하여 복잡하지 않은 전역 상태(UI 테마, 선택된 포트폴리오 등)를 관리
- 폼 상태: React Hook Form을 활용하여 사용자의 투자 성향 입력 및 백테스팅 파라미터 설정 폼을 관리

## UI/UX 및 데이터 시각화 (XAI)
- 사용자가 AI의 매매 결정 과정과 논리적 근거를 직관적으로 파악할 수 있도록 시각화된 대시보드(Dashboard) 형태의 설명 가능한 인터페이스를 반드시 구현합
- 차트 및 기술적 지표 표기를 위해 D3.js 또는 TradingView의 Lightweight Charts 라이브러리를 적극 활용
