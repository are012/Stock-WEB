# 백엔드 및 AI 개발 규약
## API 설계
- RESTful 원칙을 엄격히 따르며, OpenAPI 3.0 사양을 사용하여 프론트엔드와의 계약을 명확히 함
- 실시간 매매 시그널이나 에이전트의 상태 변화를 프론트엔드에 전달하기 위해 WebSocket을 도입
- 모든 API에는 엄격한 입력 검증을 적용하고, 통일된 형식의 오류 응답을 사용

## AI 및 멀티 에이전트 시스템 (Python 기반)
- 펀더멘털 분석(Analyst), 차트 분석(Chartist), 리스크 관리(Risk Manager) 등 전문화된 역할을 수행하는 에이전트 협업 아키텍처를 설계
- Python과 에이전트 프레임워크를 기반으로 에이전트 오케스트레이션을 구현
- LangChain과 Vector DB(ChromaDB)를 연동하여 실시간 데이터 처리 파이프라인을 구축
- RAG(검색 증강 생성) 기술을 활용해 비정형 텍스트에서 독점력과 성장성을 평가하는 '헤게모니 스코어링 모델'을 핵심 로직으로 적용

## 데이터베이스 작업 및 안정성
- Node.js 환경에서는 Prisma ORM을, Python 환경에서는 SQLAlchemy를 사용하여 데이터베이스와 통신
- 금융 데이터를 다루므로 복잡한 매매 기록이나 자산 변동 작업에는 반드시 트랜잭션을 사용하여 데이터 무결성을 보장
- 정성적 데이터의 벡터화 및 의미론적 검색을 통해 '헤게모니' 요소를 정량화하고, 이를 주가 및 거래량 기반의 기술적 지표와 결합하여 정교한 매매 시그널을 생성하는 하이브리드 알고리즘을 개발 및 데이터베이스화함
- 민감한 API 키(증권사 API, LLM API 등) 접근 및 트레이딩 실행은 별도의 보안 로그로 기록
